## О проекте

Тестовое задание для разработчика Python. Веб-приложение с API-интерфейсом и админ-панелью для управления сетью по продаже электроники.

Сеть представляет собой иерархическую структуру из трех уровней:
- Завод (уровень 0) - не имеет поставщика
- Розничная сеть (уровень 1) - поставщик - завод
- Индивидуальный предприниматель (уровень 2) - поставщик - розничная сеть

Уровень иерархии определяется автоматически на основе отношения к поставщику.

## Технологический стек

- Django 5.2.11
- Django REST Framework 3.16.1
- PostgreSQL 18+
- Poetry для управления зависимостями
- django-filter для фильтрации API

## Функциональность

### Модели данных
- Product - продукт (название, модель, дата выхода)
- NetworkNode - звено сети со следующими полями:
  * Название
  * Контакты (email, страна, город, улица, номер дома)
  * Продукты (связь Many-to-Many)
  * Поставщик (ссылка на другое звено)
  * Задолженность перед поставщиком (с точностью до копеек)
  * Время создания (автоматически)

### Админ-панель
- Удобный интерфейс для управления звеньями и продуктами
- Ссылка на поставщика на странице объекта
- Фильтр по названию города в списке звеньев
- Admin action для очистки задолженности у выбранных объектов
- Автоматическое отображение уровня иерархии

### API
- Полный CRUD для модели поставщика
- Запрет обновления поля "Задолженность" через API
- Фильтрация объектов по стране (?country=Россия)
- Поиск по названию, городу, стране
- Сортировка по различным полям

### Безопасность
- Доступ к API только для авторизованных пользователей
- Проверка активных сотрудников

## Установка и запуск

### Предварительные требования
- Python 3.12+ (рекомендуется 3.12 для стабильности)
- PostgreSQL 10+
- Poetry

### Пошаговая установка

1. Клонировать репозиторий
   git clone https://github.com/Galaxy-Voyager/electronics-network.git
   cd electronics-network

2. Установить зависимости через Poetry
   poetry install

3. Создать файл .env в корне проекта
   Содержимое файла .env:
   DB_NAME=electronics_network
   DB_USER=postgres
   DB_PASSWORD=1357
   DB_HOST=localhost
   DB_PORT=5432
   DJANGO_SECRET_KEY=your-secret-key-here-change-it
   DJANGO_DEBUG=True

4. Создать базу данных PostgreSQL
   Через psql (для PostgreSQL 18):
   & "C:\Program Files\PostgreSQL\18\bin\psql.exe" -U postgres -c "CREATE DATABASE electronics_network;"
   
   Или через pgAdmin:
   - Подключитесь к серверу (пароль: 1357)
   - Правой кнопкой на "Databases" -> "Create" -> "Database"
   - В поле "Database" введите: electronics_network
   - Нажмите "Save"

5. Применить миграции
   poetry run python manage.py migrate

6. Создать суперпользователя
   poetry run python manage.py createsuperuser
   (введите логин: admin, email: admin@example.com, пароль: придумайте сами)

7. Запустить сервер
   poetry run python manage.py runserver

## Использование

### Админ-панель
- Откройте браузер и перейдите по адресу: http://127.0.0.1:8000/admin/
- Войдите с созданными учетными данными
- Управляйте продуктами и звеньями сети через интерфейс

### API
- Откройте браузер и перейдите по адресу: http://127.0.0.1:8000/api/
- Для доступа к API необходима авторизация (кнопка "Log in" в правом верхнем углу)

## API Эндпоинты

### Звенья сети (Network Nodes)
- GET /api/nodes/ - список всех звеньев
- POST /api/nodes/ - создать новое звено
- GET /api/nodes/{id}/ - детальная информация о звене
- PUT /api/nodes/{id}/ - полное обновление звена
- PATCH /api/nodes/{id}/ - частичное обновление звена
- DELETE /api/nodes/{id}/ - удаление звена

### Продукты (Products)
- GET /api/products/ - список всех продуктов
- POST /api/products/ - создать новый продукт
- GET /api/products/{id}/ - детальная информация о продукте
- PUT /api/products/{id}/ - полное обновление продукта
- PATCH /api/products/{id}/ - частичное обновление продукта
- DELETE /api/products/{id}/ - удаление продукта

### Фильтрация и поиск
- Фильтр по стране: /api/nodes/?country=Россия
- Поиск по названию: /api/nodes/?search=Samsung
- Сортировка: /api/nodes/?ordering=name
- Сортировка по убыванию: /api/nodes/?ordering=-created_at
